Specter — VTuber Engine

Specter is a standalone, browser-based VTuber rigging engine. It lets you generate character artwork with AI, rig it with a mesh and bone system, paint weights to control how bones deform each part, and preview real-time animations — all in one interface.

About
Specter is fully self-contained. It does not require any external VTuber software. Models are saved as .specter files that package everything — textures, rig, weights, animations — into a single portable archive.

How it's organised

The editor is split into 8 modes, each shown as a tab in the top bar. You can switch between them at any time without losing work. Think of the workflow as a pipeline:

  1. Generate — Create or import your character artwork
  2. Edit — Organise layers, set opacity, remove backgrounds
  3. Mesh — Define the triangulated mesh on each layer
  4. Bones — Build a bone hierarchy and link it to parameters
  5. Weights — Paint how much each bone influences each vertex
  6. Physics — Add spring physics for hair and accessories
  7. Preview — Play animations and tweak parameters live
  8. Live — Connect a face-tracking source for real-time VTubing

AI assistance

Steps 1–5 all have AI-assisted options — you can let the AI do the heavy lifting and then fine-tune manually, or skip the AI entirely and do everything by hand. AI features require API keys in the shared .env file (Google AI or OpenAI).

The .specter File Format

A .specter file is a ZIP archive that contains your entire model — artwork, mesh data, bones, weights, animations, and physics — in one portable file.

Archive structure

yourmodel.specter ← ZIP archive
├── manifest.json ← name, version, timestamps
├── model.json ← full model data (see below)
└── textures/
├── abc123.png ← embedded layer textures
├── abc123_raw.png ← original backup (before BG removal)
└── abc123_nobg.png ← background-removed version

model.json structure

KeyTypeDescription
versionstringFormat version, currently "2.0.0"
namestringModel display name
canvasobjectWorking canvas size in pixels. Default 2048×2048
layersarrayAll layer objects (mesh parts). See Layer reference below.
bonesarrayAll bone objects forming the skeleton
weightsobjectlayerId → boneId → [weight per vertex]
parametersobjectNamed sliders that drive bone transforms (e.g. ParamAngleX)
bone_paramsarrayBindings that connect parameters to bone properties via keyframes
physics_groupsarraySpring physics chains
animationsobjectNamed animation clips with parameter tracks

Backward compatibility

Legacy avatar.specter.json models from Specter v1 are automatically detected and migrated to v2 format when you open them. The original files on disk are not modified.

Tip
You can open a .specter file with any ZIP tool (7-Zip, WinRAR, etc.) to manually inspect or extract its contents. The model.json is plain readable JSON.

Generate Mode

Use AI to generate character artwork or upload your own, then auto-rig everything in one step.
Gshortcut

The Generate panel slides out from the left of the canvas. It is the starting point for creating a new character. You can either describe a character and let AI generate the artwork, or upload an existing image.

Pipelines

AI Sheet

Generates a full sprite sheet of character parts using an image AI (Google Imagen or DALL-E). The AI then slices the sheet into individual layers and rigs them automatically. Best for brand new characters.

Upload

Upload an existing character image (PNG/JPG). The AI uses vision to detect body parts, slices them into layers, generates meshes, bones, and weights automatically. Best when you already have artwork.

Local SD

Connects to a locally-running Stable Diffusion server (A1111 compatible) to generate artwork. Requires the SD server to be running on your machine. Best for privacy or offline use.

Settings

SettingTypeDescription
Prompt text Describe the character you want to generate. Be specific about style, hair, outfit, and colors. Example: "Anime VTuber girl, silver twin-tail hair, blue eyes, white school uniform, simple expression". Not used in Upload mode.
Custom instructions text Optional rigging hints passed to the AI. Example: "Make the hair physics very bouncy" or "This character has a large hat, treat it as a separate rigid layer".
Mesh density select Controls how many mesh vertices are generated per layer.
Low — 4-vertex quad. Best for rigid or background parts.
Medium — 4×4 grid (25 verts). Good default for most parts.
High — 8×8 grid (81 verts). Best for detailed deformation.
AI adaptive — AI places extra vertices near important areas (eyes, mouth).
Bone style select Controls which bone template the AI uses.
Humanoid — Standard template (Root→Body→Neck→Head + arms, hair, eyes). Works for most VTubers.
Minimal — Only Root, Body, Neck, Head. Fast, good for simple characters.
AI custom — AI designs a bone hierarchy based on your specific layer names.
Image model select Which AI image generation model to use. Auto selects the best available model from your configured providers. Only relevant for AI Sheet and Local SD pipelines.
Chat model select Which AI language model handles rig generation, bone placement, and weight logic. Auto selects your default Google AI or OpenAI model.

Two-step flow (AI Sheet / Local SD)

  1. Click Generate — this generates and previews a concept image.
  2. Review the concept image in the preview area below the settings.
  3. Click Rig It! — this slices the image into layers, generates meshes, bones, weights, and opens the model in the editor.
Tip
If you don't like the concept, click Generate again before clicking Rig It. You can keep regenerating until you're happy with the image, then rig it.

Edit Mode

Manage your layers — reorder, rename, set opacity, and remove backgrounds.
Eshortcut

Layer Panel

The layer panel on the left shows all layers in your model, sorted from front (top) to back (bottom). Each row shows the layer thumbnail, name, and vertex count.

  • Click a layer to select it. The inspector on the right updates accordingly.
  • Drag a layer row to reorder it (changes its Z-order).
  • Eye icon — toggle layer visibility. Hidden layers are not rendered.
  • + button (panel header) — opens a file picker to import an image as a new layer.
  • Trash button (panel header) — deletes the selected layer. This cannot be undone.

Inspector — Layer Settings

Mesh Mode

Define and edit the triangulated mesh for each layer. Meshes are what makes deformation possible.
Mshortcut

Every layer needs a mesh — a set of vertices connected by triangles — to be deformed by bones. Without a mesh, a layer is just a flat image that moves rigidly.

The mesh is shown as an SVG overlay on top of the canvas. Select a layer in the left panel first, then use the toolbar tools to edit its mesh.

Toolbar Tools

Select

Click vertices to select them (shown in pink). Hold Shift to add to the selection. Selected vertices can be moved together by dragging.

Add Vertex

Click anywhere on the canvas to add a new vertex. New vertices are automatically connected to the nearest existing vertices to form triangles.

Delete Vertex

Click a vertex to delete it. All triangles that used that vertex are also removed. Remaining vertices are re-indexed automatically.

Move

Drag any selected vertex to reposition it in the canvas. If multiple vertices are selected, all move together relative to the drag distance.

Auto Mesh (Inspector)

In the right inspector, you can regenerate the mesh for the selected layer using AI or predefined grids:

DensityVerticesBest for
Low (quad)4Rigid parts: hats, accessories, simple props
Medium (4×4)25Body, head, arms — general use
High (8×8)81Face, eyes, mouth — detailed deformation areas
AI adaptivevariableAsks the AI to place extra vertices near important facial regions
Important
Regenerating the mesh on a layer resets its weights. Run Auto Weights again after changing a mesh, or manually repaint the affected weights.

Bones Mode

Create a bone skeleton that drives deformation. Bones connect to parameters to enable expressions and movement.
Bshortcut

How bones work

Bones are positioned in canvas space and form a parent-child hierarchy. When a parent bone moves, all child bones move with it. This lets you build a spine chain (Root → Body → Neck → Head) where moving the head automatically carries the eyes and mouth.

Bones themselves don't deform anything directly — they need weights (painted in Weights mode) to tell each vertex how much a bone influences it.

Toolbar Tools

Select

Click a bone (the diamond shape or its head dot) to select it. The selected bone is highlighted in cyan. Its properties appear in the inspector.

Add Bone

Click on the canvas to place a new bone. If a bone is already selected, the new bone becomes its child. Otherwise it becomes a root bone.

Delete Bone

Click a bone to delete it. Its children become unparented (root bones). You can also press Delete when a bone is selected.

Move

Drag a bone's head dot to reposition it. The bone's world position updates in real time.

Inspector — Bone Properties

PropertyTypeDescription
NametextDisplay name. This is used in the bone tree, weight painter bone list, and auto-weight heuristics.
LengthnumberVisual length of the bone in canvas pixels. Affects the end-point position (tail), and therefore where child bones attach.
RotationdegreesResting angle of the bone in degrees. 0 = pointing right. −90 = pointing up (typical for spine bones).
ColorcolorVisual color of the bone in the editor overlay. Does not affect rendering.
X / YnumberCanvas-space position of the bone's head (origin point) in pixels.

Parameters and Bone Bindings

The Parameters panel (bottom-left) shows sliders for all defined parameters. Parameters are the "inputs" that move bones:

  • ParamAngleX — Head left/right (−30 to +30 degrees)
  • ParamAngleY — Head up/down
  • ParamAngleZ — Head tilt/roll
  • ParamBreath — Breathing cycle (0 to 1)
  • ParamEyeOpenL / R — Eye open/close (0 = closed, 1 = open)
  • ParamMouthOpenY — Mouth open amount
  • ParamMouthForm — Mouth shape (−1 = frown, 0 = neutral, +1 = smile)
  • ParamBrowLY / RY — Eyebrow height

Bone-param bindings (stored as bone_params) map each parameter value to a bone property via keyframes. Auto Bones generates sensible default bindings automatically.

Weights Mode

Paint bone influence onto mesh vertices. Weights control how much each bone moves each part of a layer.
Wshortcut

Weights are stored per-vertex, per-bone as values between 0 (no influence) and 1 (full influence). At each vertex, all bone weights sum to 1.0 — the system automatically normalises them.

Vertex colors show influence level:

  • Red — High influence (close to 1.0)
  • Orange/Yellow — Moderate influence
  • Green — Low influence
  • Blue — No influence (0.0)

Workflow

  1. Select a layer in the left panel (the layer you want to paint)
  2. Select a bone in the inspector or bone panel (the bone you're painting for)
  3. Use the brush to paint on the canvas. Vertices change color to show their new weight.
  4. Repeat for other bone/layer combinations as needed.

Brush Settings

SettingTypeDescription
Radius 10–200 The size of the brush in screen pixels. Affects how many vertices are painted per stroke. The brush circle is shown as a dashed ring on the canvas.
Strength 1–100% How quickly the weight is applied per paint stroke. Low strength = gradual, high strength = fast. Applies with a falloff from the brush center.
Mode: Add mode Increases the weight of the active bone at painted vertices. The most common mode — use it to build up bone influence where you want deformation.
Mode: Subtract mode Decreases the weight of the active bone at painted vertices. Use it to remove unwanted influence or clean up edges.
Mode: Smooth mode Averages each vertex's weight with its neighbors. Removes sharp transitions in the weight map. Use it to fix artifacts and blend boundaries.

Auto Weights

The Auto Weights button in the inspector (or in the Bones mode inspector) automatically assigns weights to all layers based on:

  • Layer name heuristics — e.g. a layer named "Hair" is bound to HairL/HairR bones
  • Distance weighting — vertices closer to a bone get higher influence
  • Smoothing — weights are smoothed across neighbors to remove sharp seams

Auto Weights is a good starting point. You'll almost always want to touch up specific areas by hand afterward, especially around joints and boundaries.

Tip
Select each layer one at a time in the left panel while painting. Only the selected layer's vertex weights are shown and editable.

Physics Mode

Add spring physics to hair, tails, ribbons, and other soft accessories for natural secondary motion.
—

Physics groups apply spring simulation to chains of bones. When the head rotates, physics bones will lag behind and oscillate naturally before settling.

Physics Groups

Click Add Group to create a new physics group. Each group has:

SettingTypeDescription
Input parameter select Which parameter drives the physics chain. Typically ParamAngleX (head left/right) for hair that swings when the head turns.
Gravity 0–2 How strongly gravity pulls the physics bones downward. Higher values make the hair hang lower and fall faster.
Momentum 0–1 How much the bones resist following their target position. High momentum = sluggish, floaty. Low momentum = snappy, stiff. Typical value: 0.7–0.85.
Damping 0–1 How quickly oscillation dies down. High damping = oscillation stops fast. Low damping = bones keep swinging for a long time. Typical value: 0.1–0.2.
Note
Physics simulation runs in real time during Preview and Live modes. It is disabled in editing modes (Mesh, Bones, Weights) for stability.

Preview Mode

Play your model's animations and test how parameters affect deformation in real time.
Pshortcut

Preview mode enables the animation system and physics simulation. Use it to check that your rig looks correct before going live.

Playback

The ▶ Play / ⏸ Pause button in the bottom bar controls animation playback. The current animation name is shown next to it.

Inspector — Preview Settings

SettingTypeDescription
Animation Clip select Which animation to play. The built-in idle animation includes breathing (sine wave on ParamBreath) and occasional blinking.
Loop toggle Whether the animation loops. When off, it plays once and stops at the last frame.
Background Color color Sets the canvas background color for preview. Only visible in Preview/Live modes.
Greenscreen toggle Sets the background to pure green (#00ff00) for chroma-key compositing in streaming software like OBS. Keep the canvas window captured in OBS and key out the green.

Parameters panel (left)

In Preview mode, the Parameters panel is fully interactive. Drag any slider to manually override a parameter. This lets you test head angles, eye closing, mouth movements, and any custom parameters without a tracker. Parameters return to their animation-driven values when you release and playback is running.

Animation system

Animations are defined as tracks in model.json. Each track drives one parameter over time:

Track typeDescription
sineSmooth sine-wave oscillation. Used for breathing, floating, idle sway. Settings: speed, amplitude, offset.
blinkPeriodic blink. Eyes stay open, then briefly close every few seconds. Settings: interval (seconds between blinks).
fixedHolds a constant value. Useful for toggling layers or locking a parameter during an animation.

Layers Panel (Left)

Always visible on the left side. Shows all layers in your model sorted by Z-order (front to back).

  • Each row shows the layer thumbnail, name, type, and vertex count.
  • Click to select the layer for editing.
  • Drag rows to reorder.
  • Eye icon — show/hide the layer.
  • The + button imports an image as a new layer.
  • The trash button deletes the selected layer.

Bone Tree Panel (Left)

Shows the bone hierarchy as an indented tree. Indent level = parent depth.

  • Click a bone to select it. It becomes highlighted in cyan on the canvas.
  • In Weights mode, clicking a bone in the tree also sets it as the active paint bone.
  • The + button adds a bone at the canvas center (becomes a child of the selected bone).
  • The trash button deletes the selected bone.
  • Each bone shows its color dot so you can identify it visually on the canvas.

Parameters Panel (Left)

Shows all defined parameters as labelled sliders. These are the animation/tracking inputs that drive bone transforms.

  • In Preview mode, drag sliders to manually test poses.
  • In Live mode, sliders show real-time tracking values.
  • In other modes, sliders are for reference and can be used to test deformation.
  • The current value is shown to the right of each label.

Inspector Panel (Right)

The inspector changes based on the current mode and selection. It shows properties for the selected layer, bone, or tool settings.

  • Edit / Mesh mode → layer properties, transform, mesh tools, BG removal
  • Bones mode → bone properties, auto-rig buttons
  • Weights mode → bone selector list, brush settings, auto-weights
  • Physics mode → physics group editor
  • Preview / Live mode → animation selector, background color, greenscreen

Tracking Bridge

The Specter Tracking Bridge (tracking/bridge.py) is the interface between face/motion tracking systems and the Specter parameter system. It is designed as a separate module to keep Specter modular.

Protocol

The tracking module communicates with the editor via WebSocket at ws://localhost:8082/tracking and sends JSON messages:

{
  "type": "params",
  "params": {
    "ParamAngleX":    15.0,
    "ParamAngleY":    -5.0,
    "ParamEyeOpenL":  0.9,
    "ParamEyeOpenR":  0.9,
    "ParamMouthOpenY": 0.3
  }
}

Custom mapping

The default mapping can be overridden by passing a param_mapping dict to the TrackingBridge class:

bridge = TrackingBridge(
    backend="mediapipe",
    param_mapping={
        "head_yaw":   "ParamAngleX",
        "mouth_open": "ParamMouthOpenY",
        # ... add custom entries
    }
)
