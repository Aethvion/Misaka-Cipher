## 1. Security & Hardening

### 1.1 🔴 Fix Wildcard CORS
**File:** `core/interfaces/dashboard/server.py`  
`allow_origins=["*"]` currently allows any origin. This is safe on localhost but becomes dangerous if the server is exposed on a LAN or cloud VM.  
**Fix:** Read allowed origins from `aethvion.yaml` and default to `["http://localhost:8080"]`. Provide a `CORS_ORIGINS` environment variable for easy override.

### 1.2 🔴 Add Optional Dashboard Authentication
The dashboard has no login gate. Anyone on the same network can read memory, run agents, and see API-key-adjacent data.  
**Fix:** Add an optional single-user PIN/password auth layer (session cookie, configurable in `.env`). If `DASHBOARD_PASSWORD` is unset, auth is skipped (dev mode). Keep it simple — no user database needed.

### 1.3 🟠 Rate Limiting on API Routes
No rate limiting exists on any FastAPI route. A runaway loop or a single bad request can hammer an external API and run up costs.  
**Fix:** Add `slowapi` (or a simple in-memory token bucket) to the FastAPI app. Apply sensible limits per route class (chat: 30 req/min, forge: 5 req/min, etc.). Expose limits in `aethvion.yaml`.

### 1.4 🟠 Secrets Scanning in Forge Output
The Forge generates and registers Python code. A malicious prompt could trick it into embedding API keys or passwords in generated tool code.  
**Fix:** Run the Intelligence Firewall's scanner on generated tool source code before registration — not just on the prompt. Refuse registration if credentials are detected.

### 1.5 🟡 HTTPS / TLS Support
For LAN or cloud deployments, plain HTTP exposes chat history and tokens.  
**Fix:** Add a `--tls` flag that loads a certificate + key path from `.env` and starts Uvicorn in SSL mode. Document self-signed cert generation in the Getting Started guide.

### 1.6 🟡 Audit Log Export
Trace IDs are created per request but there's no way to export or search the audit log from the UI.  
**Fix:** Add a dedicated **Audit Log** sub-tab in the Usage page. Support filtering by date, provider, and trace ID. Add a CSV export button.

## 2. Core Architecture Improvements

### 2.1 🟠 Circuit Breaker Pattern for Providers
If Google AI returns 5 errors in a row, the system keeps trying instead of backing off.  
**Fix:** Implement a per-provider circuit breaker (closed → open → half-open states). Expose current provider health in the dashboard header.

### 2.2 🟠 Request Queue with Priority Levels
All requests are handled immediately with no queue. Long-running Forge/Factory tasks block the event loop.  
**Fix:** Add an async priority queue. Assign priority levels: `chat > memory_query > agent_task > forge_generation`. Background tasks (Forge, Factory) run in a thread pool. Show queue depth in the dashboard status bar.

### 2.3 🟠 Streaming Responses for Chat
Most chat responses return all at once. Streaming is partially in place but not consistently used.  
**Fix:** Wire up SSE/WebSocket streaming end-to-end for all providers (Google, OpenAI, Anthropic, Grok, and local GGUF). Render tokens as they arrive in the chat UI — makes the system feel dramatically faster.

### 2.4 🟡 Plugin / Extension System
New tools and agents are hard-coded. There's no way for a user to drop in a Python file and have it auto-registered.  
**Fix:** Define a simple `AethvionPlugin` interface. Anything placed in `core/plugins/` that implements the interface gets auto-discovered on startup and registered in the tool registry. Document the interface with a sample plugin.

### 2.5 🟡 Configurable Startup Checks
The launcher does basic checks but doesn't surface errors well (e.g., wrong Python version, missing API key, port already in use).  
**Fix:** Add a pre-flight check system that runs before the server starts, prints a colour-coded summary (`✅ / ⚠️ / ❌`), and aborts cleanly with actionable messages on critical failures.

### 2.6 🟢 Hot-Reload Config Without Restart
Changing `aethvion.yaml` or `providers.yaml` requires a full server restart.  
**Fix:** Add a `POST /api/config/reload` endpoint and a **Reload Config** button in the Settings tab. Use `watchdog` to optionally auto-reload on file change.

## 3. Provider & Model Layer

### 3.1 🟠 Ollama Integration
The `local_provider.py` only supports GGUF via llama-cpp-python. Ollama is far more user-friendly and widely used.  
**Fix:** Add an `OllamaProvider` that speaks to the Ollama REST API (`http://localhost:11434`). Auto-detect running Ollama instance on startup. Show available Ollama models in the model selector.

### 3.2 🟠 vLLM Integration
For users with capable GPUs who want high-throughput local inference.  
**Fix:** Add a `VLLMProvider` that speaks to the vLLM OpenAI-compatible API. Configurable base URL in `providers.yaml`.

### 3.3 🟠 Model Capability Tags
The model selector shows names but not capabilities (vision, code, context window, cost tier).  
**Fix:** Enrich `suggested_models.json` with tags: `vision`, `code`, `long-context`, `cheap`, `reasoning`. Show tags as coloured badges in the model selector UI.

### 3.4 🟡 Smart Auto-Routing Improvements
The current auto-router selects a provider but the logic is opaque.  
**Fix:** Make auto-routing explain its decision. Return a `routing_reason` string (e.g., "Selected Gemini Flash — low complexity, cost-optimised") that appears as a subtle annotation under the AI response.

### 3.5 🟡 Cost Budget Alerts
Users can accidentally run up large bills with no warning.  
**Fix:** Add a configurable `monthly_budget_usd` in `aethvion.yaml`. When the tracked spend approaches 80% and 100% of budget, show a dashboard banner. Optionally block new requests when over budget.

### 3.6 🟢 Provider Comparison Mode
Send the same prompt to two or more providers simultaneously and display responses side-by-side.  
**Fix:** Add a **Compare Mode** toggle in chat. When active, a split-pane view shows Provider A vs Provider B responses for the same query, with latency and token-cost shown for each.

## 4. Memory Tier

### 4.1 🟠 Memory Retrieval Quality Improvements
Per the docs: "retrieval improving." Semantic search in ChromaDB relies on default embeddings.  
**Fix:** Allow the embedding model to be configured (e.g., swap `all-MiniLM-L6-v2` for `nomic-embed-text` or a larger model). Add a relevance score threshold — results below the threshold are filtered out rather than returned with low confidence.

### 4.2 🟠 Memory Timeline View
The memory tab shows a flat list. Episodic memories have timestamps but no timeline visualisation.  
**Fix:** Add a **Timeline** view to the Memory tab — a chronological strip (like a git log graph) showing memory clusters by day/week. Click a cluster to expand and search within it.

### 4.3 🟡 Memory Tagging & Namespaces
All memories go into one pool. There's no way to separate "work project A" memories from "personal" memories.  
**Fix:** Add namespaces/tags to the memory schema. Let users assign a namespace per conversation thread. Memory search can then be scoped to a namespace.

### 4.4 🟡 Memory Importance Scoring
Not all stored memories are equally useful. A trivial "hello" chat shouldn't compete with a detailed architecture decision.  
**Fix:** Add an importance score (1–10) computed at storage time using the LLM. High-importance memories are retrieved with a boost; low-importance ones decay over time (a soft-delete threshold).

### 4.5 🟡 Manual Memory Editing
Users can't edit or delete individual memories from the UI.  
**Fix:** Add edit/delete buttons to each memory card in the Memory tab. Confirm deletion with a modal. Allow editing the memory text directly.

### 4.6 🟢 Memory Export / Import
No way to back up or migrate memories between installs.  
**Fix:** Add **Export All Memories** (JSON/CSV) and **Import Memories** buttons to the Memory tab. Imported memories are deduplicated against existing entries using semantic similarity.

## 5. The Factory (Agent System)

### 5.1 🟠 Agent Persistence Across Sessions
Agents are transient. A long-running task that gets interrupted has no way to resume.  
**Fix:** Add an optional `persistent=True` flag to agent specs. Persistent agents serialise their state to disk after each step. On restart, the orchestrator offers to resume incomplete tasks.

### 5.2 🟠 Multi-Agent Coordination
Agents can't currently communicate with each other or split work.  
**Fix:** Implement a simple **Agent Mesh** — agents can post messages to a shared bus (in-memory dict or Redis-optional). Add a `delegate_to_agent(agent_name, task)` tool that agents can call. Show inter-agent communication in the Agent Workspace log.

### 5.3 🟡 Agent Templates Library
Creating agents requires knowing the Aethvion naming standard and spec format.  
**Fix:** Ship a library of 10–15 pre-built agent templates (file organiser, web scraper, code reviewer, meeting summariser, etc.). Accessible from a **Templates** button in the Agent Workspace tab.

### 5.4 🟡 Agent Marketplace / Sharing
Users can't share agent specs with others.  
**Fix:** Add an **Export Agent Spec** button (downloads JSON). Add **Import Agent Spec** (upload JSON, validate, register). This sets the stage for a community marketplace.

### 5.5 🟢 Agent Visualisation
It's hard to see what an agent actually did.  
**Fix:** Add a **Task Graph** visualisation in the Agent Workspace — a node/edge diagram showing steps taken, tools called, and files touched. Uses the D3.js force graph (consistent with the existing knowledge graph visualisation).

## 6. The Forge (Tool Generation)

### 6.1 🟠 Improve Multi-Step Tool Generation
The Forge works for simple tools. Complex tools that require multiple helper functions or external imports often fail.  
**Fix:** Add a **decomposition phase** before code generation — the LLM first writes a structured plan (function signatures + docstrings), then fills in implementations. Validate each function independently before assembling the tool.

### 6.2 🟠 Tool Unit Test Generation
Generated tools have no tests.  
**Fix:** As part of Phase 4 (registration), auto-generate a pytest test file for each tool. Store it alongside the tool in the registry. Add a **Run Tests** button on each tool card.

### 6.3 🟡 Tool Versioning
Regenerating a tool overwrites the previous version.  
**Fix:** Add version numbers to tool registrations. Keep the last N versions (configurable). Show a version history dropdown on each tool card with a diff view.

### 6.4 🟡 Forge Dry Run Mode
Users can't preview a tool before it's registered.  
**Fix:** Add a **Preview** button in the Forge UI that runs Phases 1–3 and shows the generated code + security scan report without registering. The user can then click **Register** or **Regenerate**.

### 6.5 🟢 Natural Language Tool Discovery
When users ask for a tool that already exists under a different name, the Forge regenerates it instead of reusing it.  
**Fix:** Before generating, the Forge searches the registry using semantic similarity on the tool description. If a match above 0.8 cosine similarity is found, suggest the existing tool and ask "Did you mean this one?"

## 7. Web Dashboard UX

### 7.1 🟠 Global Search
With 25+ tabs and features spread everywhere, there's no way to find things quickly.  
**Fix:** Add a `Ctrl+K` / `Cmd+K` command palette — a floating modal with a fuzzy search box. Searching returns tabs, tools, agents, memory entries, and settings. Selecting navigates directly.

### 7.2 🟠 Dark / Light Theme Toggle
The dashboard is dark-only.  
**Fix:** Add a theme toggle in the top nav. Implement a CSS custom-properties-based light theme. Save preference to `localStorage`.

### 7.3 🟡 Customisable Tab Order and Pinning
With 25+ tabs, navigation is unwieldy.  
**Fix:** Allow drag-to-reorder tabs. Add a ★ pin icon to keep favourite tabs always visible. Store order/pins in `preferences.json`.

### 7.4 🟡 Keyboard Navigation
Most interactions require a mouse.  
**Fix:** Add keyboard shortcuts for the most common actions (new thread, send message, switch tab). Show a `?` shortcut reference modal accessible from the bottom of the nav sidebar.

### 7.5 🟡 Mobile-Responsive Layout
The dashboard is desktop-only.  
**Fix:** Apply responsive breakpoints at 768px. On mobile: collapse the tab list into a hamburger menu, stack panels vertically, and increase touch target sizes. The chat interface is the highest-priority page for mobile.

### 7.6 🟡 Notification System
There is no way for background tasks (agents, forge jobs) to alert the user when complete.  
**Fix:** Add a 🔔 notification bell in the top nav. Background tasks post notifications on completion (with success/failure status). Use browser `Notification` API for optional desktop notifications.

### 7.7 🟢 Dashboard Widgets / Home Page
The home screen is a static landing page.  
**Fix:** Turn it into a customisable **widget grid** — widgets include: recent chats, active agents, memory stats, system health, cost today, and a weather widget (using the existing Nexus weather integration). Users drag widgets to rearrange.

## 8. Chat & Conversation

### 8.1 🟠 Message Editing and Regeneration
Users can't edit a sent message or regenerate a response.  
**Fix:** Add an **Edit** pencil icon on hover for user messages. Editing a message re-runs inference from that point and discards subsequent messages in the thread. Add a **Regenerate** button on AI messages.

### 8.2 🟠 File & Image Attachments
No way to attach files or images to a chat message.  
**Fix:** Add a 📎 attachment button. For supported vision models (GPT-4o, Gemini 1.5, Claude 3), send the image as a vision input. For non-vision models, extract and include text content (PDFs, .txt, .md, .py files). Show thumbnails in the chat.

### 8.3 🟡 Prompt Templates Library
Users repeat the same prompt patterns (summarise, translate, explain code).  
**Fix:** Add a **/** slash-command shortcut in the chat input. Typing `/summarise` expands to a full summarisation prompt template. Ship 20 built-in templates; let users add custom ones.

### 8.4 🟡 Response Bookmarking and Starring
Useful AI responses disappear into thread history.  
**Fix:** Add a ⭐ bookmark icon on AI messages. Starred messages appear in a new **Starred** sub-tab within the chat. Optionally auto-save starred messages to the Memory Tier.

### 8.5 🟡 Conversation Branching
Sometimes you want to explore two different follow-up directions from the same point.  
**Fix:** Add a **Branch** button at any assistant message. Creates a new thread forked from that point. Branches are shown as a tree in the thread sidebar.

### 8.6 🟡 Chat Export
No way to export a conversation.  
**Fix:** Add an **Export** button per thread: Markdown (formatted), JSON (raw data), or plain text. Useful for saving research sessions, code reviews, etc.

### 8.7 🟢 AI Persona Profiles
Misaka Cipher has a personality but no way to switch it.  
**Fix:** Add a **Personas** feature: pre-configured system prompts that give the AI different personalities (tutor, code reviewer, brainstorm partner, devil's advocate). Selectable from a dropdown in the thread header.

## 9. Games Tab Expansions

The Games tab already has Blackjack, Sudoku, Checkers, Logic Quest, and Word Search. Here are expansions and new games that fit the AI-native theme.

### 9.1 🟠 AI Explains Its Moves (All Games)
AI opponents don't explain why they made a move.  
**Fix:** Add an **AI Commentary** toggle. When on, after each AI move a small speech bubble shows its reasoning ("I blocked your right flank to prevent a forced win in 3 moves"). Uses the active provider for real-time explanation.

### 9.2 🟡 Chess with AI Opponent
Chess is the canonical AI game and notably missing.  
**Fix:** Integrate `chess.js` for game logic and `chessboard.js` for the board UI. Add difficulty levels: Easy (random legal moves), Medium (minimax depth 3), Hard (call Stockfish via a tiny WASM build or the LLM for commentary). Track Elo across sessions.

### 9.3 🟡 Trivia Challenge (AI-Generated Questions)
**Fix:** A real-time trivia quiz where the LLM generates questions on-demand. Choose category (history, science, pop culture) and difficulty. AI can generate an unlimited supply so the game never repeats. Track high scores.

### 9.4 🟡 Twenty Questions (AI Guesses Your Object)
Classic party game, perfect for a conversational AI.  
**Fix:** The AI asks up to 20 yes/no questions to guess what the user is thinking. Uses chain-of-thought reasoning visible in an expandable panel. Shows confidence percentage after each answer.

### 9.5 🟡 Story Builder (Collaborative Fiction)
**Fix:** A multiplayer-style story game where the user writes one sentence, the AI writes the next, and so on. Genre selectable (fantasy, sci-fi, horror). Save completed stories to Memory. Show character/word count.

### 9.6 🟡 Debate Arena (Fun Upgrade)
The existing "Smarter Than AI" section is good — expand it.  
**Fix:** Let users pick a topic and watch two AI providers debate it (one argues FOR, one AGAINST). The user votes on the winner. Scores tracked per provider over time — a fun way to compare model reasoning styles.

### 9.7 🟢 Wordle Clone with AI Analysis
**Fix:** Classic Wordle but after each guess the AI provides a hint using its vocabulary knowledge ("None of those letters appear in common 5-letter words starting with 'cr'…"). AI can also solve it optimally if you ask.

### 9.8 🟢 Dungeon Crawler (Text RPG)
**Fix:** A simple text-based RPG where the LLM is the game master, generating rooms, enemies, and items dynamically. Player has stats (HP, attack, defence). Save/load game state to Memory. Uses ASCII art for dungeon maps.

## 10. Arena — AI vs AI Battles

### 10.1 🟠 ELO Leaderboard
Arena battles produce winners but no persistent ranking exists.  
**Fix:** Add an ELO rating system per provider/model. Track battles in `data/arena/history.json`. Show a leaderboard in the Arena tab with ratings, win/loss records, and average response quality scores.

### 10.2 🟡 Battle Categories
All battles use a single format.  
**Fix:** Add battle categories: **Reasoning** (logic puzzles), **Creativity** (story/poem), **Code** (generate a function), **Factual** (Q&A accuracy), **Speed** (first valid answer wins). Each category has a specialised judge prompt.

### 10.3 🟡 Human Judge Mode
AI judging is fast but not always trustworthy.  
**Fix:** Add a **Human Judge** option. Both AI responses are shown side-by-side without labels (blind evaluation). The user picks the better answer. Result is recorded and contributes to the leaderboard.

### 10.4 🟢 Tournament Mode
**Fix:** Run a round-robin or single-elimination tournament across all configured providers. Generates a bracket visualisation. Fun for comparing newly added models.

## 11. Code IDE App

### 11.1 🟠 Multi-File AI Context
The AI currently receives a flat file tree as context. Large projects overwhelm the context window.  
**Fix:** Add smart context selection — the AI gets the currently open file + files imported by it (parsed via AST for Python/JS). Users can manually pin additional files with a 📌 button in the file tree.

### 11.2 🟠 Git Integration
There's a git branch in the status bar but no git operations.  
**Fix:** Add basic git commands to the IDE: `status`, `diff`, `commit`, `log`, `checkout branch`. Show a **Source Control** panel (like VS Code's SCM panel). Display inline diff gutter markers in Monaco.

### 11.3 🟡 AI Code Review on Save
**Fix:** Add an optional "AI review on save" toggle. When enabled, saving a file triggers a quick AI review that posts inline comments (errors, warnings, suggestions) as annotation markers in Monaco. The review uses the configured model.

### 11.4 🟡 Integrated Terminal Improvements
The current terminal runs commands but doesn't maintain state.  
**Fix:** Make the terminal a true persistent shell session (xterm.js). Support multiple terminal tabs. Allow the AI to suggest and run shell commands with an Approve/Deny gate.

### 11.5 🟡 Extension / Plugin Support
**Fix:** Add a marketplace/library of Monaco editor plugins (Prettier formatting, ESLint, black for Python) that can be enabled per workspace from the Settings panel in the IDE.

### 11.6 🟢 Pair Programming Mode
**Fix:** The AI watches the file you're editing and suggests the next logical block of code in a ghost text overlay (similar to GitHub Copilot inline suggestions). Triggered by 2-second idle.

## 17. Developer Experience

### 17.1 🟠 Docker / Docker Compose Support
No Dockerfile exists. Setup is Windows-bat-centric.  
**Fix:** Create `Dockerfile` and `docker-compose.yml` that spin up the core suite + optional apps. Use multi-stage builds to keep image size small. Document in the Getting Started guide.

### 17.2 🟠 Linux / macOS Launch Scripts
Only `.bat` scripts exist.  
**Fix:** Create equivalent `.sh` scripts for Linux and macOS. Auto-detect platform in the scripts and choose the right launcher. Add a `Makefile` with common targets (`make start`, `make test`, `make lint`).

### 17.3 🟡 Development Server with Hot-Reload
Developers have to restart the server after every code change.  
**Fix:** Add a `--dev` flag that runs Uvicorn with `--reload` and watches `core/` for changes. Also enable detailed error pages in dev mode.

### 17.4 🟡 Interactive API Documentation (Swagger UI)
FastAPI ships Swagger docs for free but they may be disabled or not surfaced.  
**Fix:** Enable `/docs` and `/redoc` in dev mode (disable in production unless opted in). Link to API docs from the dashboard Settings tab.

### 17.5 🟡 Dependency Graph Visualisation
With 181 Python files, it's hard to understand module dependencies.

### 18.2 🟠 Provider Mock Layer for Tests
Tests that call real AI providers are unreliable and cost money.  
**Fix:** Create a `MockProvider` class that returns deterministic canned responses. All tests default to using `MockProvider`. Tests that intentionally test real providers are marked with a `@pytest.mark.live` marker and skipped in CI.

**Fix:** Execute Forge-generated tool validation in a subprocess with reduced privileges (drop filesystem write access outside `tmp/`, no network access). Use Python's `subprocess` + `resource` module limits on Linux.
